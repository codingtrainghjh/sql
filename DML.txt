'롤백 투 세이브포인트' - 지정 포인트까지 되돌리기 -> 롤백만 사용할 시 맨 처음으로 되돌아감
비정상적 종료시(ex 사고, 오류발생, 강제종료) 자동 롤백됨
수정중에는 다른 유저가 관여할 수 없음
COMMIT후에는 세이브포인트가 사라지고 롤백이 되지 않음
트랜잭션이 진행중 오류가 날 경우 오류만 취소되고 진행 상황은 남아있음
문장오류는 롤백이나 커밋으로 진행시키지 못함
두 사람이 같은 데이터에 대해 관여할 때, 읽는사람은 쓰는사람을 기다릴 필요가 없음(역도 성립)
단, 쓰는사람은 다른 쓰는사람을 기다려야 함.

어떤 직원의 연봉이 4000일때 연봉을 5000으로 업데이트하였으나 커밋을 하지 않았다.
이때 읽는 사람의 창에 나타나는 연봉은 얼마? 
-> 4000 = 업데이트 전후의 정보가 둘다 저장되어있는 상태

CONTROL파일은 큰 파일은 아니나 다른 파일을 찾는 파일로 없어지면 파일을 불러올 수 없어 큰일 남

파일업데이트 -> 임시 등록후 잠금
롤백 -> 잠금 취소후 이전 정보 복구
커밋 -> 잠금 해제 후 등록완료

DELETE는 데이터만 지울뿐 공간은 남아있음
